## Abstract

artoo is a multi-agent system for autonomous task execution on a local machine. A user submits a natural-language request; artoo decomposes it into subtasks, executes them in parallel or in sequence, validates each result against machine-checkable criteria, and replans when the result falls short — all without human intervention. The central innovation is a dual nested control loop borrowed from classical control theory: a fast inner loop (Executor + Agent-Validator) handles per-subtask correction in real time, while a slow outer loop (Goal Gradient Solver — **GGS**) computes a loss function over the full task outcome and issues mathematically grounded replanning directives. Experience from every task is encoded as Megrams and stored in the **MKCT** (Megram / Knowledge / Common Sense / Thinking) cognitive memory pyramid, which the Planner queries before each new task to exploit past successes and avoid past failures.

v0.9 closes two structural gaps in the memory system identified during testing: (1) the `IntentSlug` derivation function produced empty slugs for non-ASCII (CJK) intents, making all Chinese-language task experiences collide in one degenerate LevelDB bucket; (2) the Dreamer's upward consolidation path — LLM distillation of accumulated M-level Megrams into timeless C-level SOPs — was deferred in v0.8 and never implemented, leaving the memory pyramid write-only above the M layer.

## v0.8 Issues Fixed in v0.9

### Issue #34 — `IntentSlug` produces empty slug for all CJK intents

**Root cause**: `IntentSlug` strips every character outside `[a-z0-9]`. All CJK characters are silently dropped. Any Chinese-language intent (e.g. `"查询今天的农历日期"`) produces `space = "intent:"` — the same degenerate bucket for every Chinese-language task.

**Consequence**: All Chinese-intent terminal Megrams accumulate in one LevelDB key. The dual-channel potentials for that key become a meaningless aggregate of unrelated tasks (news lookups, calendar queries, coding questions). R2 receives `"SHOULD PREFER: Follow the same general approach"` regardless of what the current task actually is. Lessons from failures never accumulate to the threshold required for C-level promotion.

**Fix**: Use `task_id` (already ASCII snake_case, generated by R1) as the slug source instead of the raw intent string. `space = "intent:" + taskID`. The task_id is always safe as a LevelDB key segment and is unique per task class (R1 generates `today_lunar_date`, `find_largest_video`, etc.). Cross-task learning works correctly: a future task that R1 names `today_lunar_date` will query the same bucket as a prior run with the same name.

**`IntentSlug` status**: the function is retained for backward compatibility and for tools that still call it (tests, REPL display), but it is no longer used on the memory write or read hot paths.

### Issue #35 — Dreamer upward consolidation never implemented; memory pyramid is write-only above M level

**Root cause**: `runDreamer()` in v0.8 called only `gcPass()` and `trustBankruptcyPass()`. The upward consolidation step — cluster M-level Megrams above `Λ_promote` threshold, invoke LLM to distil a C-level SOP — was explicitly deferred with the comment `// Upward consolidation (LLM distillation into C-level SOPs) deferred to Phase 2`.

**Consequence**: The C-level SOP layer is always empty. `QueryC` always returns `[]`. R2 never receives articulated lessons — only raw directional potentials from the M layer, which can say "avoid" but not "avoid *because*". The memory system cannot learn across tasks in a meaningful sense: it accumulates experience but never synthesises strategy.

**Fix**: Implement `consolidationPass()` in the Dreamer cycle. The pass:
1. Scans all M/K-level Megrams grouped by `(space, entity)`.
2. For each group: computes live `M_attention` and `M_decision`.
3. Promotes to C-level when `M_attention ≥ Λ_att = 5.0` AND `|M_decision| ≥ Λ_dec = 3.0`.
4. Invokes one LLM call (`distilSOP`) to synthesise the cluster content into a single articulated rule.
5. Writes a new `Level=C, k=0.0` Megram. Marks source M/K Megrams as consolidated (sets `State="consolidated"`) so they are not re-promoted on the next cycle.

## R5 — Shared Memory (v0.9)

### The MKCT Pyramid (unchanged structure, upward flow now active)

               ^                                                      |
               |              /:::::::::::::\                         |
               |             /    [ T ]      \                        |
  Immutable    |            /   THINKING      \                  Immutable
  Agent Laws   |           /___________________\                (No Demotion)
  k = 0.0      |          /                     \                     |
               |         /        [ C ]           \                   |
  Promotion    |        /     COMMON SENSE         \              Demotion
  M_att >=5.0  |       /   (SOPs & Constants)       \             M_dec < 0.0
  |M_dec|>=3.0 |      /________________________________\          k reverts to 0.05
  k = 0.0      |     /                                  \               |
               |    /            [ K ]                    \             v
  Clustering & |   /           KNOWLEDGE                   \       Time Decay
  Lazy Eval    |  / (Task Cache & Local Context)             \     g(Δt) = e^(-k*Δt)
  k > 0        | /______________________________________________\        |
               |/                                                \       v

### Tag Convention Change (v0.9)

| Event type | Space | Entity | v0.8 | v0.9 |
|---|---|---|---|---|
| Terminal macro-event | `"intent:<slug>"` | `"env:local"` | slug = `IntentSlug(intent)` | slug = `taskID` |
| Action micro-event | `"tool:<name>"` | `"target:<value>"` | unchanged | unchanged |

Using `taskID` as the slug ensures:
- Every distinct task class gets its own LevelDB bucket (R1 generates stable, descriptive task IDs).
- CJK intents produce the same quality of slug as English intents.
- Cross-task learning works: repeated runs of `today_lunar_date` accumulate in `"intent:today_lunar_date"`.

### Dreamer — Offline Consolidation Engine (v0.9)

Runs as a background goroutine on a 5-minute timer. Never blocks the operational hot path.

**Full three-phase cycle (all active in v0.9)**:

**Phase 1 — Physical Forgetting (GC)** *(unchanged from v0.8)*:
- M/K entry where live `M_attention < Λ_gc = 0.1` → hard DELETE from LevelDB.

**Phase 2 — Trust Bankruptcy** *(unchanged from v0.8)*:
- C-level entry where live `M_decision < 0.0` → strip time immunity; k reverts to 0.05; demoted to K level.

**Phase 3 — Upward Consolidation** *(new in v0.9)*:
- Scan all M/K-level Megrams grouped by `(space, entity)`.
- For each group with `M_attention ≥ Λ_att = 5.0`:
  - If `M_decision ≥ Λ_dec = 3.0` → distil **Best Practice** (C-level, σ=+1.0).
  - If `M_decision ≤ −Λ_dec = −3.0` → distil **Constraint** (C-level, σ=−1.0).
- Invokes one LLM call per qualifying group to synthesise a generalised rule from the cluster's `content` fields.
- Writes new `Megram(Level=C, k=0.0, sigma=±1.0)` to LevelDB.
- Marks source entries as `State="consolidated"` so they skip future consolidation scans.
- Already-existing C-level Megrams for the same `(space, entity)` are updated via `RecordNegativeFeedback` + new write if the new distillation contradicts the old one.

**Distillation prompt** (sent to LLM):
```
You are synthesising accumulated task experience into a reusable rule.

Space: <space>
Entity: <entity>
Signal: <"Best Practice" | "Constraint">
Valence: <M_decision>

Source experiences (newest first):
<content_1>
<content_2>
...

Write ONE concise rule (≤2 sentences) that captures the generalised lesson from these experiences.
The rule must be actionable: a Planner reading it should know exactly what to do or avoid.
Output ONLY the rule text, no preamble.
```

### `memory.New` Signature Change

```go
// v0.8
func New(b *bus.Bus, dbPath string) *Store

// v0.9
func New(b *bus.Bus, dbPath string, llmClient *llm.Client) *Store
```

The LLM client is required for Phase 3 distillation. If `nil`, Phase 3 is skipped and the Dreamer runs only GC + Trust Bankruptcy (graceful degradation to v0.8 behaviour).

## Key Invariants (additions for v0.9)

| Invariant | Enforced by |
|---|---|
| Terminal Megram `space` uses `taskID`, not `IntentSlug(intent)` | GGS `writeTerminalMegram` |
| Planner queries `"intent:" + taskID` not `IntentSlug(intent)` | Planner `queryMKCTConstraints` |
| Dreamer consolidation only promotes when `M_att ≥ 5.0 AND |M_dec| ≥ 3.0` | `consolidationPass` thresholds |
| Consolidated source Megrams are marked `State="consolidated"` to prevent re-promotion | `consolidationPass` post-write |
| Distillation LLM call is skipped when `llmClient == nil` (graceful degradation) | `distilSOP` nil-check |
| Each `(space, entity)` group produces at most one new C-level Megram per Dreamer cycle | `consolidationPass` per-group dedup |
