## 2026-03-04 ‚Äî Memory v0.9: pyramid fix, Dreamer consolidation, Ctrl+C bug

**Participants**: Hari, opencode (Claude Sonnet 4.6)

### Context
Investigated why `today_lunar_date` task took 240s total with R3 burning 124.7s in round 2.

### Findings

**R3 was not the root cause.** R2 planned a call to `opendata.baidu.com/api.php?resource_id=39043` which returned 89 almanac entries with all date fields as `None`. R3 faithfully executed R2's plan ‚Äî tried 9 different Python parsing approaches against the same broken endpoint. R3 was the victim; R2 was the source.

**Why R2 didn't learn from memory:** two structural bugs in the v0.8 memory system.

**Bug #93 ‚Äî `IntentSlug` CJK degeneracy**
`IntentSlug()` strips all non-ASCII characters. Any Chinese-language intent (e.g. `"Êü•ËØ¢‰ªäÂ§©ÁöÑÂÜúÂéÜÊó•Êúü"`) produces `space = "intent:"` ‚Äî the same degenerate LevelDB bucket for every Chinese-language task. All Chinese-intent terminal Megrams accumulated together, making dual-channel potentials meaningless noise. The lesson from `today_lunar_date` failing would have been misfiled under `"intent:"` forever.

Fix: use `task_id` (already ASCII snake_case, generated by R1) as the space tag. `"intent:" + taskID`. R1 produces stable, descriptive IDs (`today_lunar_date`, `find_largest_video`) regardless of the input language.

**Bug #94 ‚Äî Dreamer upward consolidation never implemented**
`runDreamer()` called only `gcPass()` and `trustBankruptcyPass()`. The upward consolidation step ‚Äî cluster M-level Megrams above threshold, invoke LLM to distil a C-level SOP ‚Äî was deferred in v0.8 with a `// deferred to Phase 2` comment. The C-level layer was permanently empty. R2 could only receive a directional signal ("avoid this") but never an articulated reason ("avoid this *because* the Baidu API returns null fields for 2026 dates").

Fix: implemented `consolidationPass()` ‚Äî scans M/K Megrams grouped by `(space, entity)`, promotes clusters where `M_att ‚â• 5.0 AND |M_dec| ‚â• 3.0` via one LLM `distilSOP()` call per qualifying group. `memory.New()` gains a `llmClient` parameter; `nil` degrades gracefully to v0.8 behaviour.

**Bug ‚Äî terminal Megram content was vague (found same session)**
Even after fixing #93 and #94, the Megram `Content` field for `accept`/`success`/`abandon` states was just the user-facing summary string (`"‚úÖ Task completed within convergence threshold. 1 subtask(s) completed."`) ‚Äî no tool names, no commands. The Dreamer LLM distillation would have produced rules like *"this worked"* rather than *"use zhdate Python library via shell"*.

Fix: new `buildTerminalContent()` helper in `ggs.go` extracts `(tool, target)` pairs from `ToolCalls` of matched (accept/success) or failed (abandon) outcomes and embeds them in `Content`:
- accept/success ‚Üí `"Succeeded. Tools: shell:python3 -c 'import zhdate; ...'. Result: ‰ªäÂ§©ÂÜúÂéÜÊòØÊ≠£ÊúàÂçÅÂÖ≠"`
- abandon ‚Üí `"Failed. Tools tried: shell:curl http://opendata.baidu.com/api.php?resource_id=39043. Gap: API returned null fields"`

**Bug #95 ‚Äî Ctrl+C stuck when idle**
The signal handler goroutine was declared before `rlCh` and `rlResult`. When `taskCancel == nil` (idle) it did nothing ‚Äî the comment said "readline's ErrInterrupt handles exit" but readline is only blocked on `Readline()` once the main loop reaches `readLine()`. Between task completion and the next `readLine()` call, Ctrl+C was silently consumed and lost, making artoo appear stuck.

Fix: moved signal handler goroutine declaration after `rlCh`. When idle, injects `rlResult{err: readline.ErrInterrupt}` directly into `rlCh` so the main loop's normal "first press warns, second press exits" path fires correctly.

**Also added in this session:**
- R2 memory query now logs at `INFO` level (was `DEBUG`, invisible in `debug.log`): `space`, `entity`, `sops`, `attention`, `decision`, `action`, `constraints_injected`, and the full constraint string when non-empty.

## 2026-03-04 (session 2) ‚Äî Memory content gap: MKCT pyramid worse than old flat file

**Participants**: Hari, opencode

### Finding
Hari observed that the old rough memory system (flat `memory.json` + keyword scan + direct content injection into R2 prompt) performed *better* than the new MKCT pyramid. This is correct and has a precise cause.

**What the old system injected into R2:**
```
SHOULD PREFER (prior successes ‚Äî these approaches worked):
  - Used mdfind then shell ls -lS. Found /Downloads/agentic-shell-demo.mov (85M).
MUST NOT (prior failures ‚Äî do not repeat these approaches):
  - Baidu Open Data API resource_id=39043 returns 89 entries, all date fields None.
```
Concrete tool names, concrete commands, concrete outcomes. R2 could reason and copy.

**What the new MKCT system injected:**
```
SHOULD PREFER (memory: this approach worked well for similar tasks):
  - Follow the same general approach that succeeded previously.
```
One generic vacuous sentence. The actual Megram `Content` field ‚Äî even after our earlier fix to enrich it with tool names ‚Äî was thrown away by the dual-channel convolution math in `QueryMK`. `QueryMK` produces only `{Attention, Decision, Action}` ‚Äî a scalar direction, not content. And C-level SOP promotion requires `M_att ‚â• 5.0` (‚âà5 runs), so R2 gets nothing actionable until well after cold start.

### Root cause
`calibrateMKCT` only read from `QueryC` (C-level SOPs, always empty until Dreamer promotes) and `QueryMK` (direction-only potentials). The raw Megram `Content` was unreachable from R2.

### Fix
Added `QueryRecent(ctx, space, entity, n)` to `MemoryService` interface and `Store`. Returns up to `n` most recent M/K-level non-consolidated Megrams newest-first ‚Äî bypassing the convolution math to surface raw content directly.

Updated `calibrateMKCT` to inject three layers in priority order:
1. **C-level SOPs** (Dreamer-distilled rules) ‚Äî highest authority, empty until warm
2. **Recent M/K Megram content** (raw past experience) ‚Äî available from 2nd run; contains `"Succeeded. Tools: shell:python3 -c 'import zhdate'..."` or `"Failed. Tools tried: shell:curl http://opendata.baidu.com/..."`
3. **Dual-channel potential heuristic** ‚Äî only shown when layers 1+2 are both empty (cold start fallback)

Layer 3 is now suppressed when layers 1 or 2 have content, eliminating the vacuous "Follow the same general approach" sentence.

**Result**: from the 2nd run onward, R2 sees concrete tool names, commands, and outcomes ‚Äî same quality as the old flat-file system, but now embedded in the pyramid architecture so it will continue to improve via Dreamer consolidation over time.
